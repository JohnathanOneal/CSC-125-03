Python as a Calculator

You already know basic math symbols from school. Python uses the same ones, plus a few extras.

Think of it like a calculator that remembers its answers. You type an expression, Python evaluates it, and you can store the result in a variable for later.

  • The familiar four: + - * / (add, subtract, multiply, divide)
  • Three new ones: // (floor division), % (remainder), ** (power)

The first four work exactly like you expect. The interesting ones are the last three - let's focus on those.

The Pizza Problem: Floor Division and Remainder

Imagine you have 17 slices of pizza and 5 people. Two questions come up:

  • How many full slices does each person get? That is floor division: 17 // 5 gives 3
  • How many slices are left over? That is the remainder (modulo): 17 % 5 gives 2

These two operators always work as a pair. Together they answer: "how many whole groups, and what is left?"

Where You See This in Real Life

Floor division and remainder show up everywhere once you know to look for them:

  • Time conversion: 135 minutes is how many hours and minutes? 135 // 60 = 2 hours, 135 % 60 = 15 minutes
  • Making change: How many quarters fit in 87 cents? 87 // 25 = 3, with 87 % 25 = 12 cents left
  • Even or odd: Any number % 2 gives 0 if even, 1 if odd

The pattern is always the same: you are splitting a total into groups and asking what remains.

Two Kinds of Division

A common source of confusion: Python has two division operators that behave differently.

Regular /

Works like a calculator. Always gives a decimal (float), even when it divides evenly.

10 / 5 gives 2.0 (not 2)

Use when you need precise answers.

Floor //

Rounds down and drops the decimal. Gives a whole number.

10 // 3 gives 3 (not 3.33)

Use when you need whole groups.

Tip: if your math result unexpectedly has .0 at the end, you probably used / when you meant //, or vice versa.

Order of Operations: PEMDAS

Python follows the same rules you learned in math class. It does not just read left to right.

This means 2 + 3 * 4 is 14, not 20. Python multiplies first (3 * 4 = 12), then adds (2 + 12 = 14).

If that is not what you intended, parentheses override everything: (2 + 3) * 4 gives 20.

Section 2 · Lecture

The Average Trap

One of the most common beginner mistakes involves calculating an average. Suppose you have three test scores and want their average:

wrong_avg = 72 + 85 + 68 / 3    # 179.67 - only 68 gets divided
right_avg = (72 + 85 + 68) / 3  # 75.0   - all three are added first

Without parentheses, Python divides only the last number by 3 (because division has higher precedence than addition). The parentheses force the addition to happen first.

Rule of thumb: when in doubt, add parentheses. They never hurt and they make your intent clear to anyone reading your code.

Asking Yes-or-No Questions: Comparisons

Sometimes you do not need a number - you need a yes or no answer. Python calls these True and False.

Think of comparisons like questions you ask Python:

  • "Is this person old enough to vote?" → age >= 18
  • "Did the student pass?" → score >= 70
  • "Are these two names the same?" → name1 == name2
  • "Is the price within my budget?" → price <= budget

Each of these gives back True or False. There are six comparison operators: == != < > <= >=

Section 2 · Lecture

Saving Answers for Later

Just like you store numbers in variables, you can store True/False answers in variables. This makes your code read like English:

can_vote = age >= 18
is_passing = score >= 70

Now instead of repeating the comparison everywhere, you just check can_vote or is_passing. Naming your booleans with is_, has_, or can_ makes them read naturally.

Next week, we will use these True/False values to make decisions in our code with if statements. This is the foundation for that.

Keeping a Running Tally: Assignment Operators

Imagine you are keeping score in a game. Every round, you need to update the score based on what it already is. Python has shortcuts for this:

  • score += 10 means "add 10 to the current score" (same as score = score + 10)
  • score -= 5 means "subtract 5 from it"
  • score *= 2 means "double it"

Think of it like a cash register: you start at zero and keep adding items. The running total updates each time. We will use these heavily when we learn loops.

Two Common Pitfalls

= vs ==

Single equals stores a value.
Double equals asks a question.

Think: one equals sign is a command ("put this here"). Two equals signs is a question ("are these the same?").

Strings are not numbers

input() always gives you a string. You cannot do math on a string.

"25" + 10 causes an error. You need to convert first: int("25") + 10

This came up in Week 2 - same rule applies here.

Summary

  • Python has 7 arithmetic operators - the 4 familiar ones plus //, %, and **
  • Floor division and modulo split things into whole groups and leftovers
  • Regular division (/) always gives a decimal; floor division (//) gives a whole number
  • PEMDAS controls order - use parentheses when in doubt
  • Comparisons ask yes/no questions and give True or False
  • Store booleans in descriptive variables (is_passing, can_vote)
  • Assignment operators (+=, -=) update a variable based on its current value
