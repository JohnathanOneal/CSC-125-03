Section 2 · Lecture

The Coffee Shop Problem

Imagine you track daily coffee orders. One variable per order? That works for three, but not for fifty or five hundred. A list is like a clipboard with a numbered sheet: one place to hold all related items.

order1 = "latte"
order2 = "mocha"
order3 = "espresso"

Instead of a separate variable for each, a list keeps them together:

orders = ["latte", "mocha", "espresso"]

Section 2 · Lecture

Anatomy of a List

  • Square brackets [] are the container, commas separate items
  • Lists can hold any type: strings, numbers, booleans, or a mix
fruits = ["apple", "banana", "cherry"]
count = len(fruits)  # 3

len(fruits) returns 3, telling you how many items are inside.

An empty list [] is a blank container, ready to be filled.

Section 2 · Lecture

Every Item Has an Address

Like apartments in a building, every item has a number starting at 0. The index measures distance from the start. The first item is 0 steps away.

  • Positive: count from the front (0, 1, 2)
  • Negative: count from the back (-1, -2, -3)
name = fruits[0]    # "apple"
last = fruits[-1]   # "cherry"

Section 2 · Lecture

Taking a Portion

Slicing is like cutting slices from a loaf of bread. You choose where to start and stop.

  • Syntax: list[start:end]. The end index is not included (the knife cuts before it).
  • Omit start to begin from the front, omit end to go to the back.
numbers = [10, 20, 30, 40, 50]
middle = numbers[1:4]  # [20, 30, 40]

Section 2 · Lecture

Lists are Like a Whiteboard

A whiteboard lets you erase and rewrite at any time. Lists work the same way. Change any item by assigning to its index.

fruits[1] = "mango"  # was "banana"
print(fruits)  # ["apple", "mango", "cherry"]

This ability to change is called mutability. Later we will see tuples, which are like a printed poster: once printed, the content is fixed.

Section 2 · Lecture

Your List Toolkit

Lists come with built-in tools:

  • .append(item) adds to the end (writing the next line on your clipboard)
  • .remove(item) removes the first match (erasing a specific entry)
  • .pop() removes and returns the last item (pulling the bottom card from a stack)
  • len(list) counts items
cart = ["milk", "eggs"]
cart.append("bread")  # ["milk", "eggs", "bread"]

Section 2 · Lecture

The Sealed Envelope

A tuple is like a sealed envelope. Once sealed, the contents cannot be changed. Created with parentheses: point = (10, 20)

  • Indexing works the same way: point[0] returns 10
  • You cannot add, remove, or change items. Python raises an error if you try.
  • Why use them? When data must stay fixed: coordinates, colors, dates.
point = (10, 20)
x, y = point  # x is 10, y is 20

Unpacking lets you split a tuple into separate variables in one line.

Section 2 · Lecture

Choosing the Right Container

Lists []

  • Mutable (changeable)
  • Use when items may change
  • Shopping cart, game scores, to-do items
  • Many methods: append, remove, sort

Tuples ()

  • Immutable (fixed)
  • Use when data should stay locked
  • Coordinates, colors, dates
  • Only count and index

Quick rule: if items will change, use a list. If they should stay fixed, use a tuple.

Planning Before Building

Before building a house, an architect draws a blueprint. Pseudocode is your blueprint for code. It describes your program's logic in plain, structured English.

  • No syntax rules. No colons or indentation required. Just clear, numbered steps.
  • It helps you think through the logic before worrying about Python's rules.

Example: 1. GET the user's name becomes name = input("Enter name: ")

Section 2 · Lecture

From English to Python

Start each step with an action verb: GET, SET, CALCULATE, DISPLAY, IF, FOR. Number your steps for order. Use language a non-programmer would understand.

1. GET the list of test scores
2. CALCULATE the average
3. IF average >= 70
4.     DISPLAY "Passing"
5. ELSE
6.     DISPLAY "Not passing"

Pseudocode is about thinking, not typing. Get the logic right first, then translate to Python.

Summary

  • Lists [] store multiple items in one variable
  • Indexing starts at 0 (positive) or -1 (from the end)
  • Slicing [start:end] extracts a portion (end not included)
  • Lists are mutable (whiteboard), tuples are immutable (sealed envelope)
  • .append(), .remove(), .pop() modify lists
  • Pseudocode plans your logic in plain English before coding
