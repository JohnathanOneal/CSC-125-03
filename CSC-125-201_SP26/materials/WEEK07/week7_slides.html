Slicing with a Step

Last week we sliced with list[start:end]. There is a third option: the step. Think of stepping stones across a pond. A step of 2 means you land on every other stone.

  • list[start:end:step] picks every step-th item
  • Omit start and end to step through the whole list
  • A negative step walks backward through the list
numbers = [10, 20, 30, 40, 50, 60]
evens = numbers[::2]   # [10, 30, 50]

Reversing a List

Sometimes you need data in the opposite order: newest first instead of oldest, highest to lowest, or simply reading backward. Python gives you two approaches.

  • [::-1] creates a new reversed copy (the original stays the same)
  • .reverse() flips the list in place (changes the original, returns None)
letters = ["a", "b", "c", "d"]
backward = letters[::-1]  # ["d", "c", "b", "a"]

Choose based on whether you still need the original order.

Why Sort?

Imagine a bookshelf where books are placed randomly. Finding a specific title means checking every single book. Sorting puts them in order so you (or your program) can find things faster and spot patterns like the highest or lowest values.

  • Sorted data is easier to read, compare, and analyze
  • Finding the top 3 scores? Sort and take the last three.
  • Looking for duplicates? Sorted data puts them side by side.

Python provides two ways to sort, and choosing the right one matters.

Section 2 · Lecture

.sort() vs sorted()

This is one of the most common beginner mistakes. Both sort data, but they work very differently.

.sort()

  • Changes the list in place
  • Returns None
  • The original order is gone
scores = [88, 72, 95]
scores.sort()

sorted()

  • Creates a new sorted list
  • Returns the new list
  • The original is preserved
scores = [88, 72, 95]
ranked = sorted(scores)

Never write x = scores.sort(). That stores None, not a sorted list.

Is It in the List?

Before searching a filing cabinet, you first ask: "Is the file even in here?" The in keyword answers that question for lists. It returns True or False.

  • in checks if a value exists in a list
  • not in checks the opposite
  • Combine with if to make decisions based on membership
menu = ["coffee", "tea", "juice"]
has_coffee = "coffee" in menu  # True

Finding Where and How Many

Once you know an item is in the list, you might need to know where it sits or how many times it appears. Two methods handle this.

  • .index(value) returns the position of the first match (like finding a page number in a table of contents)
  • .count(value) returns how many times the value appears
  • .index() raises an error if the value is missing, so check with in first
colors = ["red", "blue", "red", "green"]
pos = colors.index("blue")   # 1
qty = colors.count("red")    # 2

Growing and Combining Lists

You already know .append() adds one item. But what if you need to add an item at a specific position, or merge two lists together?

  • .insert(index, value) places an item at a specific position (like cutting in line)
  • .extend(other_list) adds all items from another list to the end (merging two guest lists into one)
  • The + operator also joins lists, but creates a new one instead of modifying the original
tasks = ["email", "report"]
tasks.insert(0, "coffee")  # ["coffee", "email", "report"]

Copies: Shallow Traps

If you write copy = original, both names point to the same list. Changing one changes the other, like two name tags on the same person.

  • .copy() or [:] creates an independent copy
  • Without a proper copy, edits to one variable affect the other
original = [1, 2, 3]
safe_copy = original.copy()  # independent

Strings and Lists

A sentence is just words joined by spaces. Python can split a string into a list of parts, or join a list of parts back into a string. These two operations are like taking apart a LEGO set and rebuilding it.

  • .split(separator) breaks a string into a list of pieces
  • separator.join(list) glues a list of strings back into one
words = "hello world python".split()
# ["hello", "world", "python"]

Section 2 · Lecture

Lists Inside Lists

A list can hold any type of data, including other lists. Think of a spreadsheet: each row is a list, and the whole table is a list of rows.

  • First index selects the inner list (the row)
  • Second index selects the item within it (the column)
  • Useful for grids, tables, and organizing multi-category data
grades = [[85, 90], [78, 92], [95, 88]]
first_student = grades[0]      # [85, 90]
specific = grades[0][1]        # 90

Summary

  • Step slicing [::2] picks every Nth item. [::-1] reverses.
  • .sort() changes the list in place (returns None). sorted() returns a new list.
  • in and not in test membership. .index() finds position. .count() tallies occurrences.
  • .insert() adds at a position. .extend() merges lists.
  • .copy() or [:] creates an independent duplicate.
  • .split() and .join() convert between strings and lists.
  • Nested lists hold lists inside lists, accessed with double indexing.
