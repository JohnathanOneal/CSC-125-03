The PB&J Challenge

Here is a jar of peanut butter, a jar of jelly, a loaf of bread, and a knife.

Your task: write me instructions to make a peanut butter and jelly sandwich.

The catch: I will follow your instructions exactly as written. I have no common sense. I will not assume anything you do not explicitly state.

This is exactly how a computer reads your code - literally, one instruction at a time, with zero assumptions.

4
Section 3 · Lecture

Attempt 1: "Put Peanut Butter on the Bread"

What went wrong?

  • The bread is still in the bag
  • The jar is still sealed
  • No one said to use a knife
  • "Put" could mean anything

We assumed things that were never stated. The literal instruction-follower did exactly what we said.

Section 3 · Lecture

Attempt 2: "Open the Jar, Spread It on the Bread"

Better, but still wrong:

  • "Spread it" - spread what? The jar itself?
  • We never said to use a knife to scoop
  • We never said how much
  • We never said which side of the bread

Every unstated assumption is a potential failure point.

Section 3 · Lecture

What the PB&J Teaches Us

Every failure was a missing check: a condition we assumed but never stated.

  • IF the bread is in a bag → take it out first
  • IF the jar is closed → remove the lid
  • IF you need to spread → pick up the knife first
  • IF one side is done → move on to the jelly

These "IF" checks are conditional logic. A computer needs every condition spelled out explicitly, just like our literal sandwich-maker.

Today we learn to write these checks in Python.

Section 3 · Lecture

The if Statement

The if keyword tests a condition. If the answer is True, the indented block runs. If False, Python skips it entirely. Think of it like a bouncer checking your ID at the door.

age = 18
if age >= 18:
    print("You can vote")
  • The condition line must end with a colon :
  • The body is indented by 4 spaces
  • Code after the block (unindented) always runs regardless of the condition
8
Section 3 · Lecture

if-else: A Fork in the Road

Sometimes you need two paths: one for True, one for False. Like a fork in the road: you must take one path or the other. Never both, never neither.

number = 7
if number % 2 == 0:
    print("Even")
else:
    print("Odd")
  • else has no condition. It catches everything the if missed
  • else also needs a colon and an indented body
  • Exactly one block always runs
9
Section 3 · Lecture

if-elif-else: The Vending Machine

A vending machine checks buttons top to bottom. You press one button, you get one item. It stops checking after the first match. That is how elif works.

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
else:
    grade = "C or below"

Order matters: check from highest to lowest. The first True condition wins and all remaining branches are skipped.

10
Section 3 · Lecture

Combining Conditions: and, or, not

Sometimes one check is not enough. Python lets you combine conditions:

  • and: both must be True (need a ticket AND be 18+ to enter)
  • or: at least one must be True (weekend OR holiday = day off)
  • not: flips the value (door is NOT locked, so you can enter)
if age >= 16 and has_license:
    print("You can drive")

Back to our sandwich: "IF the jar is closed AND you have hands → twist the lid." Multiple conditions, one decision.

11
Section 3 · Lecture

Decisions Within Decisions

Sometimes one answer leads to another question, like a flowchart. You ask "do you have a ticket?" and only then ask "how old are you?"

if has_ticket:
    if age >= 18:
        price = 12.00
    else:
        price = 8.00

Each level adds 4 more spaces. Tip: keep nesting to 2 levels. Deeper nesting gets hard to read and debug.

Section 3 · Lecture

Two Common Mistakes

= vs ==

Single equals stores a value.

Double equals asks a question.

Think: one = is a command ("put this here"). Two == is a question ("are these the same?").

elif order

Always check the highest threshold first.

Wrong: if score >= 60 catches a score of 95 before you ever check for 90.

Right: start with if score >= 90, then work down.

13

Summary

  • if runs code only when a condition is True
  • if-else provides a two-way choice. Exactly one block runs
  • if-elif-else handles multiple options. First True wins
  • and, or, not combine multiple conditions
  • Nested if statements handle decisions within decisions
  • Every condition line ends with a colon, body indented 4 spaces
  • Like the PB&J: spell out every condition. A computer assumes nothing
