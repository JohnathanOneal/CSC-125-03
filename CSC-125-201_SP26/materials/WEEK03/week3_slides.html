Section 1 · Context

Why Learn the Fundamentals?

Personal example: I want to build a mobile app with a React Native frontend. I plan to use AI to help me build it. But before I do any of that, I am working through a tutorial like javascript.info/variables to learn the basics on my own.

The reason is simple. If I skip straight to AI, I cannot tell when it gives me something wrong. I cannot design the system. I cannot spot bad patterns. The AI becomes useless if I do not have the knowledge to evaluate what it produces.

Same idea here. We build up the fundamentals first so that when you do use AI tools, you know enough to work with them instead of just copying and hoping.

Section 2 · Lecture

What Are String Methods?

Think of a string like a piece of text written on paper. Methods are tools you can use on that text - a pair of scissors to trim whitespace, an eraser to swap one word for another, a stamp to change the case.

You call a method using dot notation: the string, a dot, then the tool name. For example, .strip() trims extra spaces, .upper() makes everything uppercase, and .title() capitalizes each word.

The key rule: methods give you back a new string. They do not change the original. If you do not save the result, it disappears.

cleaned = messy_text.strip()  # save the result

Section 2 · Lecture

Chaining and Replacing

Since each method returns a new string, you can chain them together - like an assembly line where each step cleans the text a little more.

clean_name = "  alice JOHNSON  ".strip().title()
# Result: "Alice Johnson"

Reading left to right: first strip the spaces, then capitalize each word. Each method works on the result of the one before it.

The .replace() method swaps one piece of text for another. Think of "find and replace" in a word processor:

fixed = msg.replace("teh", "the")  # fix a typo

Section 2 · Lecture

Indexing: Characters Have Addresses

Every character in a string has a numbered position, like houses on a street. The first house is always number 0 (not 1 - this is a quirk of programming).

You use square brackets to visit a specific address: word[0] is the first character, word[3] is the fourth.

You can also count from the end using negative numbers: word[-1] is always the last character, no matter how long the string is.

Section 2 · Lecture

Slicing: Grabbing a Section

Indexing gets one character. Slicing gets a range - like highlighting a section of text to copy it.

The syntax is [start:end], where the start is included but the end is not. Think of it as "start here, stop just before there."

Shortcuts: leave out the start to begin from the front ([:5]), leave out the end to go to the back ([6:]).

Section 2 · Lecture

F-Strings: Filling in the Blanks

Think of f-strings like a fill-in-the-blank sentence. You write the sentence with placeholders in curly braces, and Python fills them in with your variables.

The old way

Gluing pieces together with + is messy and error-prone:

"Hi " + name + ", age " + str(age)

The modern way

F-strings read like a natural sentence:

f"Hi {name}, age {age}"

You can also put expressions inside the braces: f"Tax: ${price * 0.08:.2f}". The :.2f controls decimal places.

Section 2 · Lecture

When Things Go Wrong: Reading Errors

Error messages look scary, but they are actually helpful clues. Python is telling you exactly what went wrong and where. Read them like a detective reads evidence:

  • Last line: The error type and description - what happened
  • Arrow (---->): The line number - where it happened
  • The lines above: The path Python took to get there
---> 1 result = "hello" + 5 ← 2. Location (which line)
TypeError: can only concatenate str (not "int") to str ← 1. Error type + 3. Description

Start from the bottom and work up. The error type alone often tells you what to fix.

Section 2 · Lecture

Debugging: Your New Superpower

Debugging is not about being smart enough to avoid errors. It is about having a process for finding them. Think of it like being a mechanic - you do not guess, you run diagnostics.

  • Step 1: Read the error message (last line first - that is the diagnosis)
  • Step 2: Go to the line number the arrow points to
  • Step 3: Add print() statements above that line to see what values your variables actually hold
  • Step 4: Test with simple inputs before trying complex ones

Every programmer debugs constantly. The difference between a beginner and an expert is how systematically they do it.

12

Summary

  • String methods are tools for text - .strip(), .upper(), .lower(), .title(), .replace()
  • Methods return new strings - always save the result in a variable
  • Indexing ([0]) grabs one character; slicing ([start:end]) grabs a range
  • F-strings (f"Hello {name}") are fill-in-the-blank sentences
  • Error messages are clues, not punishments - read from the bottom up
  • Debug systematically: read the error, find the line, inspect with print()
