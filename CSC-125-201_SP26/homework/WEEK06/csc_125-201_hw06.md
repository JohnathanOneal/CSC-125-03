## Pseudocode Guidelines

Good pseudocode reads like a recipe: clear, numbered steps anyone could follow without knowing Python. Start each step with an action verb:

| Verb | Meaning | Example |
|------|---------|---------|
| GET / ASK | Receive input | GET the user's name |
| CREATE / SET | Store a value or make a list | CREATE a list of 5 scores |
| CALCULATE | Perform a computation | CALCULATE the average of the scores list |
| FIND | Locate a value in data | FIND the maximum value in the list |
| DISPLAY | Show output | DISPLAY the total and average |
| IF / ELSE | Make a decision | IF average >= 70, DISPLAY "Passing" |
| FOR EACH | Repeat for every item | FOR EACH score in the list, check if below 60 |

**Be specific:** "Add all 5 scores and divide by 5" is better than "calculate average."

**Use clear conditions:** "If total spending is greater than the budget" is better than "check the budget."

**Avoid being too detailed:** "CREATE a variable to store the total" is better than "Initialize a float variable x = 0.0."

## Sample Design Document: Coffee Order Calculator

### Section 1: Overview

This program helps a coffee shop customer calculate their order total including tax. The user enters prices for 3 drinks, and the program calculates the subtotal, adds 7% tax, displays the final total, and identifies the most expensive drink. It helps people see their total before ordering and stay within budget.

### Section 2: Requirements

**Inputs:** Price of drink 1, 2, 3 (numbers).

**Calculations:** Subtotal (sum of list), tax (subtotal x 0.07), total (subtotal + tax), most expensive (max of list).

**Outputs:** Subtotal, tax, total, most expensive price, savings suggestion if most expensive exceeds $6.

### Section 3: Pseudocode

```
1. DISPLAY "Welcome to Coffee Order Calculator"
2. CREATE an empty list called prices
3. ASK user "Enter price of drink 1:" and add to the prices list
4. ASK user "Enter price of drink 2:" and add to the prices list
5. ASK user "Enter price of drink 3:" and add to the prices list
6. CALCULATE subtotal as the sum of all values in the prices list
7. CALCULATE tax as subtotal times 0.07
8. CALCULATE total as subtotal plus tax
9. DISPLAY subtotal, tax, and total
10. FIND the maximum value in the prices list, store as max_price
11. DISPLAY "Most expensive drink: $" followed by max_price
12. IF max_price is greater than 6
13.     DISPLAY "Consider a smaller size to save money"
14. DISPLAY "Thank you for your order"
```

### Section 4: Test Cases

**Test 1 - Normal order:** Inputs: $4.50, $3.25, $5.00. Expected: Subtotal $12.75, Tax $0.89, Total $13.64, Most expensive $5.00, no suggestion.

**Test 2 - Expensive drink:** Inputs: $3.00, $7.50, $4.00. Expected: Subtotal $14.50, Tax $1.02, Total $15.52, Most expensive $7.50, suggestion displayed.

**Test 3 - Boundary:** Inputs: $6.00, $6.00, $6.00. Expected: Subtotal $18.00, Tax $1.26, Total $19.26, Most expensive $6.00, no suggestion (not greater than $6).

### Section 5: Reflection

The most challenging part was deciding how to handle finding the most expensive drink. I chose to find the maximum value in the list rather than comparing each price individually, which made the pseudocode cleaner. I structured the program to collect all inputs first, then do all calculations, then display results. This mirrors how a real transaction works. If I expanded this, I would allow any number of drinks instead of exactly three.
