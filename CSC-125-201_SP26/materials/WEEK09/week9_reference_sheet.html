Basic While Loop

Repeats code as long as the condition is True. The condition is checked only at the top, not during the body.

count = 1
while count <= 5:
    print(count)
    count = count + 1  # Must update to avoid infinite loop

For vs While

Use for whenUse while when
You have a list or sequenceYou do not know how many repetitions
You know the count: range(10)You are waiting for a condition to change
Processing each itemUser input, searching, games

Condition Phrasing

The condition describes when to keep going, not when to stop.

You wantWrongRight
Stop at 5while count == 5while count < 5
Stop when emptywhile len(items) == 0while len(items) > 0
Stop on "quit"while answer == "quit"while answer != "quit"

break and continue

KeywordEffectUse when
breakExit the loop entirelyFound what you need, user typed "quit"
continueSkip rest of current iterationSkip invalid data, move to next item

With continue in a while loop, update the loop variable before the continue to avoid infinite loops.

Input Validation Pattern

valid = False
while not valid:
    response = get_input()  # Get user input
    if meets_requirements(response):
        valid = True
    else:
        print("Invalid. Try again.")

Sentinel Value Pattern

value = get_first_input()
while value != -1:        # -1 is the sentinel
    process(value)
    value = get_next_input()

Reading Tracebacks (Bottom-Up)

1. Last line: error type and description. 2. Line above: the code that failed. 3. Above that: file and line number.

try/except

# Catch a specific error
try:
    number = int(user_input)
except ValueError:
    print("Not a valid number.")

# Catch multiple errors
try:
    result = int(val) / int(other)
except ValueError:
    print("Not a number.")
except ZeroDivisionError:
    print("Cannot divide by zero.")

# Catch any error (debugging only)
try:
    risky_code()
except Exception as e:
    print(f"Error: {e}")

5-Step Debugging Process

1. Reproduce the error. 2. Read the traceback bottom-up. 3. Hypothesize what went wrong. 4. Test with a print statement. 5. Fix one thing, then re-run.
