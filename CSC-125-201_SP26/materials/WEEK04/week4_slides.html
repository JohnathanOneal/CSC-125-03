Floor Division & Modulo

Floor division (//) gives the whole number part. Modulo (%) gives the remainder.

# Convert total minutes to hours and minutes
total_minutes = 135
hours = total_minutes // 60    # 2
minutes = total_minutes % 60   # 15
print(f"{hours}h {minutes}m")  # "2h 15m"
# Even/odd check - the result is True or False
number = 14
is_even = number % 2 == 0      # True

number = 7
is_even = number % 2 == 0      # False

Division Types Compared

True Division /

Always returns a float, even when dividing evenly:

10 / 5     # 2.0
10 / 3     # 3.3333...
7 / 2      # 3.5
1 / 4      # 0.25

Floor Division //

Returns an int for int inputs (rounds down):

10 // 5    # 2
10 // 3    # 3
7 // 2     # 3
1 // 4     # 0

Use / when you need precision. Use // when you need a whole number.

Operator Precedence

Python follows PEMDAS rules. Operations higher in the table are evaluated first:

# Multiplication happens before addition
result = 2 + 3 * 4       # 14  (not 20)

# Exponents happen before multiplication
result = 2 * 3 ** 2      # 18  (not 36)

Parentheses for Clarity

Use parentheses to control order and make your intent obvious:

# Calculate the average of three temperatures
temp1 = 72
temp2 = 68
temp3 = 75
average = (temp1 + temp2 + temp3) / 3    # 71.666...

# Apply tax to a price
price = 49.99
tax_rate = 0.08
total = price * (1 + tax_rate)           # 53.9892

Even when not strictly required, parentheses improve readability:

# Both produce the same result, but the second is clearer
result = a + b * c / d
result = a + ((b * c) / d)

Boolean Expressions

Comparison results can be stored in variables for later use:

age = 20
can_vote = age >= 18          # True
is_teen = 13 <= age <= 19     # False (chained comparison)
temperature = 98.6
has_fever = temperature > 100.4    # False

price = 29.99
is_affordable = price < 50.00     # True

Boolean variables are useful for tracking the state of a condition. We will use them with conditional logic next week.

Shorthand operators update a variable using its current value:

OperatorEquivalentExample

score = 0

score += 50     # score is now 150
score *= 2      # score is now 300
score -= 75     # score is now 225

Common Mistakes

= vs ==

# Assignment (sets a value)
x = 10

# Comparison (checks equality)
x == 10    # True

= stores a value. == checks if two values are equal.

Precedence & Types

# Precedence trap
result = 10 + 5 * 2   # 20, not 30

# Type mixing
age = "25"
check = age > 18  # TypeError

Convert strings to numbers before comparing: int(age) > 18

Summary

  • Seven arithmetic operators: + - * / // % **
  • Floor division (//) returns whole numbers; modulo (%) returns remainders
  • True division (/) always returns a float
  • PEMDAS governs operator precedence; use parentheses for clarity
  • Six comparison operators: == != < > <= >=
  • Comparisons return True or False and can be stored in variables
  • Assignment operators (+=, -=, etc.) update variables in place
  • Watch out for = vs == and type mismatches
